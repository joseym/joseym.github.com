AngularJS is ...

  • an MVCish Framework
  • What HTML would have been if designed for Applications
  • Extensibility - plays well with other libraries and easily overwritten.

Model

The model represents your applications data.

Data sources, API requests, databases, hashes, arrays ...

Controller

The Controller introduces the model to the view, passing on data and logic to and from the user.

Form submission, API returns...

View

Representation of your data, or what the user interacts with.

AngularJS

is Different

  • Two-Way Data Binding
  • Opinionated - There's generally a right way and a wrong way.
  • Feels like your just writing HTML
  • Does almost ALL the heavy lifting
  • No required dependancies to get started
  • JQuery built in. ( Actually its jqlite, but still... )
